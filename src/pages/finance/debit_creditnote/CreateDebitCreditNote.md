
# CreateDebitCreditNote — Backend Schema Checklist

> **Purpose:** Frontend ↔ Backend contract verification for Credit Note and Debit Note creation.
> All endpoints are relative to `http://localhost:8000`.

---

## 1. GET Endpoints (data fetching)

| Hook | Endpoint | Response expected by FE |
|---|---|---|
| `useTenderIds` | `GET /tender/gettendersid` | `{ data: [{ tender_id, tender_project_name, tender_location: { city, state }, _id }] }` |
| `useVendors(tenderId)` | `GET /permittedvendor/getvendor/:tenderId` | `{ permitted_vendors: [{ vendor_id, vendor_name }] }` OR `{ data: { permitted_vendors: [...] } }` |
| `useContractors(tenderId)` | `GET /contractor/getbytender/:tenderId` | `{ contractors: [...] }` OR `{ data: [...] }` — each item: `{ contractor_id, contractor_name }` |
| `useNextCNNo` | `GET /creditnote/next-no` | `{ cn_no: "CN/25-26/0001" }` |
| `useNextDNNo` | `GET /debitnote/next-no` | `{ dn_no: "DN/25-26/0001" }` |
| `usePayableBills` | `GET /finance/payable-bills?supplier_id=&supplier_type=&tender_id=` | See §4 below |
| `useMaterials(tenderId)` | `GET /material/list/:tenderId` | `{ data: [{ description, category, unit, total_budgeted_qty, total_received_qty }] }` |
| `useCNList` | `GET /creditnote/list` | `{ data: [...] }` |
| `useDNList` | `GET /debitnote/list` | `{ data: [...] }` |

## 2. POST — Create Credit Note

**Endpoint:** `POST /creditnote/create`

```json
{
  "cn_no": "CN/25-26/0001",
  "cn_date": "2026-03-27",

  "reference_no": "REF-001",
  "reference_date": "2026-03-20",
  "location": "Mumbai, Maharashtra",

  "sales_type": "Local",
  "adj_type": "Against Bill",
  "tax_type": "GST",
  "gst_percent": 18,
  "rev_charge": false,

  "supplier_type": "Vendor",
  "supplier_id": "V-0012",
  "tender_id": "TND-2025-001",
  "tender_ref": "<mongo _id of tender>",
  "tender_name": "Highway Project Phase 1",

  "bill_no": "PB/25-26/0043",

  "round_off": 0.50,
  "amount": 25000.00,

  "entries": [
    { "dr_cr": "Cr", "account_name": "V-0012 – ABC Suppliers", "debit_amt": 0, "credit_amt": 25000 },
    { "dr_cr": "Dr", "account_name": "Cement OPC 53 Grade",    "debit_amt": 25000, "credit_amt": 0 }
  ],

  "narration": "3 bags cement returned — damaged on delivery",
  "status": "pending"
}
```

### Field notes — Credit Note

| Field | Type | Required | Notes |
|---|---|---|---|
| `cn_no` | string | **yes** | Auto-generated by `/creditnote/next-no` |
| `cn_date` | string (ISO date) | **yes** | |
| `reference_no` | string | no | Omitted if empty |
| `reference_date` | string (ISO date) | no | Omitted if empty |
| `location` | string | no | Auto-filled from tender location |
| `sales_type` | `"Local" \| "Interstate" \| "Export" \| "SEZ" \| "Exempt"` | **yes** | |
| `adj_type` | `"Against Bill" \| "Advance Adjustment" \| "On Account"` | **yes** | |
| `tax_type` | `"GST" \| "NonGST" \| "Exempt"` | **yes** | |
| `gst_percent` | number | no | Only sent when `tax_type === "GST"` and value is set |
| `rev_charge` | boolean | **yes** | |
| `supplier_type` | `"Vendor" \| "Contractor"` | **yes** | |
| `supplier_id` | string | **yes** | `vendor_id` or `contractor_id` |
| `tender_id` | string | **yes** | Human-readable tender ID |
| `tender_ref` | string | no | MongoDB `_id` of the tender document |
| `tender_name` | string | no | Display name of the tender |
| `bill_no` | string | no | Linked payable bill number |
| `round_off` | number | **yes** | `0` when balanced; positive = Dr heavy, negative = Cr heavy; capped at ₹1 |
| `amount` | number | **yes** | Total note value (gross) |
| `entries` | array | **yes** | Min 1 entry with non-empty `account_name`; must balance within ₹1 |
| `entries[].dr_cr` | `"Dr" \| "Cr"` | **yes** | |
| `entries[].account_name` | string | **yes** | Supplier row: supplier label; material rows: material description |
| `entries[].debit_amt` | number | **yes** | `0` when side is Cr |
| `entries[].credit_amt` | number | **yes** | `0` when side is Dr |
| `narration` | string | no | |
| `status` | `"draft" \| "pending"` | **yes** | Set by Save as Draft / Save as Pending button |

## 3. POST — Create Debit Note

**Endpoint:** `POST /debitnote/create`

Same shape as Credit Note **plus** one extra field, and different key names for doc number/date:

```json
{
  "dn_no": "DN/25-26/0001",
  "dn_date": "2026-03-27",
  "service_amt": 5000.00,

  "reference_no": "REF-002",
  "reference_date": "2026-03-20",
  "location": "Mumbai, Maharashtra",

  "sales_type": "Local",
  "adj_type": "Against Bill",
  "tax_type": "GST",
  "gst_percent": 18,
  "rev_charge": false,

  "supplier_type": "Contractor",
  "supplier_id": "C-0004",
  "tender_id": "TND-2025-001",
  "tender_ref": "<mongo _id of tender>",
  "tender_name": "Highway Project Phase 1",

  "bill_no": "WB/25-26/0012",

  "round_off": 0,
  "amount": 12500.00,

  "entries": [
    { "dr_cr": "Dr", "account_name": "C-0004 – XYZ Contractors", "debit_amt": 12500, "credit_amt": 0 },
    { "dr_cr": "Cr", "account_name": "Steel TMT 500D 12mm",       "debit_amt": 0, "credit_amt": 12500 }
  ],

  "narration": "Penalty for 5-day delay in supply @ ₹2500/day",
  "status": "pending"
}
```

### DN-specific fields

| Field | Type | Required | Notes |
|---|---|---|---|
| `dn_no` | string | **yes** | Auto-generated by `/debitnote/next-no` |
| `dn_date` | string (ISO date) | **yes** | |
| `service_amt` | number | no | Service portion of the debit note; omitted if blank |

> All other fields are identical to the Credit Note payload above.

## 4. GET — Payable Bills (Linked Bill selector)

**Endpoint:** `GET /finance/payable-bills`
**Query params:** `supplier_id`, `supplier_type`, `tender_id`

**Response expected by FE:**
```json
[
  {
    "_id": "<mongo id>",
    "bill_no": "PB/25-26/0043",
    "bill_type": "Purchase Bill",
    "bill_amount": 50000,
    "amount_paid": 25000,
    "balance_due": 25000,
    "paid_status": "partial",
    "due_date": "2026-04-15"
  }
]
```

> Only unpaid / partially-paid bills are shown. `paid_status` values consumed by FE: `"unpaid"` | `"partial"`.

| Action | Endpoint | Body |

| Approve Credit Note | `PATCH /creditnote/approve/:id` | none |

## 6. GET — Material List (Voucher Entry dropdown)

**Endpoint:** `GET /material/list/:tenderId`

**Response expected by FE:**
```json
{
  "data": [
    {
      "description": "Cement OPC 53 Grade",
      "category": "MT-BL",
      "unit": "Bags",
      "total_budgeted_qty": 500,
      "total_received_qty": 120
    }
  ]
}
```

> FE uses `description` as both the dropdown label and the `account_name` value sent in voucher entries.
> `category`, `unit`, `total_budgeted_qty`, `total_received_qty` are not used in this module but are present in the response.

---

## 7. Voucher Entry — Accounting Logic (FE-enforced)

| Note Type | Supplier row `dr_cr` | Material row `dr_cr` |
|---|---|---|
| Credit Note (CN) | **Cr** (auto-set, locked) | Dr (user-editable) |
| Debit Note (DN) | **Dr** (auto-set, locked) | Cr (user-editable) |

- Supplier entry `account_name` is **auto-filled** from the selected supplier label (`vendor_id – vendor_name`).
- Material entry `account_name` is selected from `/material/list/:tenderId` — value stored is the material `description`.
- Balance check: `|totalDr - totalCr| < ₹0.005` → balanced. Up to ₹1 difference → treated as round-off and sent as `round_off` field.

---

## 8. Frontend Validation Summary

| Rule | Where enforced |
|---|---|
| `doc_no`, `doc_date`, `amount` are required | yup schema |
| Tender must be selected | `onSubmit` guard |
| Supplier must be selected | `onSubmit` guard |
| At least one entry with non-empty `account_name` | `onSubmit` guard |
| Dr/Cr difference must be ≤ ₹1 | `onSubmit` guard |
| `amount` must be positive number | yup schema |
| `service_amt` ≥ 0 (DN only) | yup schema |
